# Git Cheatsheet

A comprehensive, searchable reference for common Git commands and workflows, designed for quick access and developer productivity.

## üöÄ Why Use Git Cheatsheet?

Managing version control can be complex. The **Git Cheatsheet** tool provides a curated list of essential Git commands, categorized by their function, to help you navigate your development workflow with ease.

### Developer-Centric Features
- **Categorized Commands**: Commands are logically grouped into sections like Setup, Staging, Branching, Remotes, and History.
- **Real-Time Search**: Instantly filter commands and descriptions to find exactly what you need. The search matches against both the command syntax and its explanation.
- **One-Click Copy**: Quickly copy any command to your clipboard. The interface provides immediate visual feedback ("Copied!") upon success.
- **Privacy-Focused**: All searching and filtering happens entirely in your browser. No data is sent to any server.

## üõ†Ô∏è Key Features

The tool organizes Git commands into the following functional categories:

- **Setup & Configuration**: Commands for setting up your identity and initializing repositories.
- **Staging & Committing Changes**: Essential commands for managing your staging area and recording changes.
- **Branching & Merging**: Tools for navigating branches, merging code, and rebasing, including modern alternatives like `git switch`.
- **Working with Remotes**: Commands for interacting with remote repositories (fetch, pull, push, and remote branch management).
- **Viewing History & Undoing Changes**: Utilities for viewing logs, comparing commits, and reverting changes.

## üìñ How to Use

1. **Access the Tool**: Navigate to the [Git Cheatsheet](https://tools.lavx.hu/tools/git-cheatsheet) page.
2. **Search**: Use the search bar at the top to filter commands. For example, type "delete" to see all commands related to deleting branches or files.
3. **Browse Categories**: Scroll through the categorized sections to find the command you need.
4. **Copy Command**: Click the **Copy** button next to any command to save it to your clipboard.
5. **Execute**: Paste the command into your terminal. Be sure to replace any bracketed placeholders (e.g., `[branch-name]`, `[url]`, or `[commit]`) with your specific project details.
